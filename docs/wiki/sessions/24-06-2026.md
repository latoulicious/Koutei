# Summary

Wired the real **PS (Physical Strength / 体力)** constants found in R-002, unblocking
F-002 at the code level and resuming the project from ON HOLD. Web-only change; the
Go solver core is numeric-agnostic and untouched.

# Files Touched

- `web/src/state.ts` — `DEFAULTS` placeholders → real `SpaceshipConst.json` values.
- `web/src/payload.ts` — new `moodSkillLine` helper (mirrors `primarySkillLine`).
- `web/src/main.ts` — `addOperator` defaults `moodLine` to the `physical_power` line.
- `web/src/payload.test.ts` — `moodSkillLine` test.
- Docs flipped ON HOLD → active: `docs/wiki/README.md`, `docs/wiki/data.md`,
  `PLAN.md`, `docs/wiki/resolutions.md` (R-002), `koutei-next-step` memory.

# Previous Behavior

`DEFAULTS = 100/100/20/15` (fabricated placeholders). New operators started with
`moodLine: null` (no aura until the user picked one). Project parked as blocked.

# New Behavior

`DEFAULTS = { stamina: 10000, staminaMax: 10000, drainBase: 12, regen: 20 }` sourced
from `Niesc-F/EndfieldTableCfg` `SpaceshipConst.json` (`maxPhysicalStrength`,
`basePhysicalStrengthCostRate`, `basePhysicalStrengthRecoveryRate`). New operators
default `moodLine` to their `physical_power` (PS-recovery) line, else null. Values
remain per-operator editable.

# Reason For Change

R-002 located the constants (the 2026-06-21 hunt failed on a naming mismatch — the
resource is PS, modelled on the Spaceship system, not "mood/stamina"). This applies
its approved "Next-session wiring" list.

# Risks

- localStorage: DEFAULTS apply to newly added operators only; existing saved rosters
  keep the old `100/100/20/15` until re-added (per-operator editable — acceptable).
- Primary/mood line overlap: 5/26 operators (avywenna, lifeng, alesh, estella,
  catcher) have `physical_power` as line 1 and no `manufacture_efficiency`, so both
  `primarySkillLine` and `moodSkillLine` default to line 1. Benign — the solver
  assigns one slot per operator-slice (no double-count); user-editable default.
  Pre-existing `primarySkillLine` `skills[0]` fallback, not introduced here.
- Solver contract / `/optimize` shape / determinism: unchanged (numeric pass-through).
