`DEFAULTS = 100/100/20/15` (fabricated placeholders). New operators started with

resource is PS, modelled on the Spaceship system, not "mood/stamina"). This applies

- Solver contract / `/optimize` shape / determinism: unchanged (numeric pass-through).

# Summary

Pinned the PS rate time unit (R-002 item 4). The desc-id route was a dead end (it's a
percentage, not a unit); adopted **per-minute** by inference and rescaled `DEFAULTS` to
per-hour slices (×60). Investigation against `Niesc-F/EndfieldTableCfg`.

# Files Touched

- `web/src/state.ts` — `DEFAULTS` `drainBase 12→720`, `regen 20→1200` (per-hour = ×60).
- `docs/wiki/data.md`, `docs/wiki/resolutions.md` (R-002 item 4), `docs/wiki/README.md`,
  `PLAN.md` — record the finding + decision, close the open caveat.

# Previous Behavior

`drainBase 12`, `regen 20` (raw source rate). On `staminaMax 10000` that drains in ~833
slices — fatigue effectively inert over any realistic horizon. Unit flagged unpinned.

# New Behavior

`drainBase 720`, `regen 1200` (source per-minute ×60, stored per 1-hour slice). Drain
full ≈13.9 h, regen full ≈8.3 h — fatigue is active over a 24 h horizon, so rotation
and the zero-stamina prune actually engage.

# Reason For Change

Desc id `-2816445264583474700` = "Mood Regen +12%" (a %, no unit). `SpaceshipConst`
sibling durations are in seconds (86400 = 1 day); only per-minute fits the PS timescale.
User chose to adopt per-minute now rather than leave the caveat open.

# Risks

- The unit is **inferred, unconfirmed** — an authoritative settle-interval would live in
  game logic, not the table dump. Documented as such.
- Behavior change: solver outputs shift (operators now tire) — but relative ordering is
  unchanged (rates scale together); `/optimize` contract + determinism untouched.
- localStorage: existing saved rosters keep old per-op values until re-added.

# Notes

`vitest` 4/4 green, `tsc --noEmit` clean.
