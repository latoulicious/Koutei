# Summary

Removed the `synergyCombo` term (F-003 ‚Üí R-003): a per-room additive bonus baked into
the domain at Phase 1 (PLAN ¬ß2 rule-3 "synergy evaluator") that models a co-placement
mechanic Endfield does not have. Confirmed by the user. End-to-end removal across the
solver core, HTTP payload, SPA, tests, and docs.

# Files Touched

- Go: `internal/domain/{rules,station}.go`, `internal/solver/{solver,branch_and_bound}.go`,
  `internal/interfaces/http/optimize.go` + the four Go test files.
- Web: `web/src/{state,main,payload}.ts`, `web/src/ui/stations.ts`, `web/src/payload.test.ts`.
- Docs: `PLAN.md`, `docs/wiki/{domain,known-constraints,data}.md`,
  `docs/wiki/modules/{domain,api,web}.md`, `docs/wiki/decisions/0001-numeric-payload.md`,
  findings.md (F-003), resolutions.md (R-003).

# Previous Behavior

`RoomEfficiency(skillBonuses, synergyCombo) = 1.0 + ő£ skillBonus + synergyCombo`.
`Station.SynergyCombo` / `synergy_combo` in the payload; a SYNERGY input in the SPA;
the B&B bound added `1.0 + synergyCombo` per room.

# New Behavior

`RoomEfficiency(skillBonuses) = 1.0 + ő£ skillBonus`. No synergy field anywhere; SYNERGY
input gone; B&B bound adds base `1.0` per room. Solver output for any input that used a
non-zero synergy now drops that inflated term.

# Reason For Change

The `+ synergyCombo` tail had no in-game basis ‚ÄĒ a Phase-1 design assumption never
validated. Only that tail was fabricated; `1.0 + ő£ skillBonus` is real.

# Risks

- **Breaking payload change** (protected contract) ‚ÄĒ but backward/forward-compatible:
  Go ignores unknown JSON fields (no `DisallowUnknownFields`), and old `localStorage`
  stations keep an ignored `synergyCombo` key. No migration needed. Internal,
  single-consumer (SPA ‚ÜĒ own backend), so no external client breaks.
- Determinism preserved (`RoomEfficiency` still pure). B&B bound stays admissible
  (1.0 ‚Č§ prior 1.0+synergy), so optimality is unaffected.

# Notes

`gofmt -l` clean, `go vet ./...` clean, `go test ./...` green, `vitest` 4/4 green,
`tsc --noEmit` clean.
